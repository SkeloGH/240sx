Creating a TunerStudio Project

TunerStudio Projects:

To present raw binary data found in the controller in an easy to read and edit fashion, TunerStudio must relate the information in several key files. To accomplish this TunerStudio uses a project. This project will tie together the firmware ini file(s), your settings, any inc files specific to your setup along with your dashboards and preferences and settings for TunerStudio.

Note: TunerStudio does have the capability to open an msq without a project, this handy feature allows you to quickly view settings from any msq you may have down loaded or gotten from a friend. However, TunerStudio does this by creating a temporary project and will not enable communications. You should always create a project if you are going to connect to and configure a controller.

Creating a Project

A project is created in few simple steps. Start by opening the "New Project" Dialog.
From the main menu:
File --> "New Project"
or
Press CTRL+N
The "New Project" Dialog will appear.

Page 1:

  • Name - You can give the project any name using characters allowed by your operating system. The Project Folder will also take on this name.
  • Project Directory - This does not normally need to be changed. By default your project will be created in:
    1. On Windows: "My Documents\TunerStudioProjects\
    2. On Linux or OS X - ~/TunerStudioProjects/
  • ECU Definition - Selecting the ECU Definition can be accomplished in 3 ways, Detect, Select from drop down or by selecting "Other" The Detect Process is the preferred approach, however if you do not have a controller or the detect process fails due to a port hang, you can use one of the alternative methods to selecting the ECU Definition file.
    1. Detect - Preferred Approach. If you have a controller available you can connect to it, power the controller up and click "Detect". TunerStudio will identify the firmware on your controller and select the appropriate ECU Definition file (ini) for your firmware if TunerStudio has it. If TunerStudio does not have the needed file, you will be prompted to browse to it through a File Dialog. This ini would have been provided with your firmware.
    2. Select from the list. You must know the firmware signature, this is not always the same as the firmware version. The signature does not represent the firmware version, you must know what the firmware serial signature is. If you know the signature, look in the drop down list to see if TunerStudio already has it. If you do not see the signature for your firmware, you will need to use approach 3 "Other"
    3. Other. - This method lets you specify the ini file you want to use, this is typically the one that came with your firmware. If you are using non mainstream firmware TunerStudio may not have the appropriate ini file. So, click "Other", the the "..." Button. When the File Dialog appears, browse to the ini file provided with your firmware.

Page 2 (Project Settings):

  • Settings - The settings available on this list will vary based on your firmware. Verify that what they are set to matches your set up. When unsure, the default value is likely fine.

Page 3 (Comm Settings):

  • Comm Settings - Select the correct port and baud for your computer. If you successfully detected the firmware these will already be set to the proper values. Clicking Test will check to see if you do have good communication with the controller, you will receive a Success or Failure response.
  • Note: MS1 is always 9600 baud, MS2 is always 115200. If not set to the correct baud you will not get a successful response.

Page 4 (Dash Board Selection):

  • Drop Down Slection - The dashboards packaged with TunerStudio will be listed in the drop down. Select any that you would like to try. If using the registered version, this can be changed later or you can add additional tabs for many more dashboards..
  • Other - If you have another dash you may have downloaded or created, you can click "Other" and browes to the dash file. If the dash was not created on the same firmware you may need to correct OutputpuChannels on any gauges that have a red slash through it.
  • Note: The Lite edition will always load the "Default" dash will alway be loaded on project open.

Finish:

  • Click "Finish" - and your Project will be created with the input you provided.
