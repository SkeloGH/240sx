Audit Review Warnings

Errors and Warning messages when loading tune settings - During the load of any tune settings file (msq), A series of validations are performed to insure all settings are valid to the constraints defined in the ECU Definition file by the Firmware developers and that the settings you are loading are of the same units as those for your project.
There is a distinct difference between errors and warnings, if the message is a warning, it is informational, a value was corrected and loaded. An Error is a critial problem that should be investigated further.
Warnings with version 1.14 -

Starting with version 1.13 of TunerStudio, there are some additional data validations performed when loading an msq. This includes checking all values when loaded from an msq to make sure they are in range, i.e. >= min and <= max as defined in the firmwares ini. If a value is found out of range a warning is issued and the value is set to the closest valid value. i.e. if greater than max, it will be set to max.

However this now means that you will get a difference report when TunerStudio connects to the controller as the values no longer match the settings in your controller.
- If you select Send TunerStudio Settings, then the invalid values will be corrected and you should see nothing more.
- If you select use Controller Settings, then TunerStudio will take the invalid settings from the controller and next time you load the msq and connect to the controller, you will repeat this scenario.

If you are opening a project for the first time, or have just changed firmware, settings you are getting a warning on are basically not used, so nothing to worry about. Those are common warning messages the first time you load an msq.
